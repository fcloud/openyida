# 钉钉 AI 解决方案中心 — 面向 SA 的方案生产系统

> 目标不是再做一个方案展示门户，而是做成 SA 从客户需求到可演示 Demo 的 AI 方案作战台。

## 一、产品定位

钉钉 AI 解决方案中心面向所有钉钉 SA、解决方案架构师、行业专家和生态交付同学，提供从客户需求识别、行业方案匹配、悟空 Prompt 生成、OpenYida 一键搭建样板应用，到客户版交付材料生成的完整工作流。

一句话定位：

**让 SA 输入客户需求，快速生成可演示、可交付、可复用的钉钉 AI 解决方案。**

核心价值：

| 对象 | 价值 |
| --- | --- |
| 一线 SA | 少找资料，少从零写方案，快速准备客户拜访和演示 |
| 行业专家 | 把专家经验沉淀为可复用方案包、Prompt、Demo 模板 |
| 交付团队 | 方案边界、应用结构、交付清单更标准，减少返工 |
| OpenYida | 把 AI 生成的方案变成宜搭里真实可运行的应用 |
| 悟空 | 承担方案理解、生成、改写、话术、材料生产和质检 |

## 二、关键判断

现有「AI 宜搭体验中心」更像样板间，核心是展示应用能力和入口。新的中心需要面向 SA 的真实工作流，核心是生产方案、演示方案和沉淀方案。

设计原则：

1. **需求优先**：入口从「客户要解决什么问题」开始，而不是从「有哪些产品功能」开始。
2. **AI 优先**：每个方案都能被悟空理解、改写、生成话术和交付材料。
3. **可执行优先**：每个重点方案都要尽量有 OpenYida 一键搭建路径。
4. **资产化优先**：方案、Prompt、Demo、PPT、交付清单都作为版本化资产管理。
5. **SA 效率优先**：默认高信息密度，适合检索、比较、复用和现场准备。

## 三、核心用户场景

### 3.1 客户拜访前

SA 输入客户行业、规模、当前系统、会议纪要或商机描述，AI 自动输出：

- 客户需求摘要
- 推荐行业方案
- 推荐钉钉能力组合
- 可复用 Demo
- 客户版演示话术
- 需要客户补充的信息

### 3.2 客户现场演示前

SA 选择方案后，一键生成：

- 宜搭样板应用
- 表单、流程、报表、权限、连接器清单
- 演示数据
- 演示路径
- 常见异议处理

### 3.3 客户确认方向后

SA 将客户反馈输入 AI，生成：

- 需求差异分析
- PRD 草稿
- 交付计划
- 验收清单
- 项目风险提示

### 3.4 方案沉淀

SA 或行业专家提交方案，中心通过 AI 质检：

- 是否有明确行业和客户对象
- 是否有业务价值
- 是否有可执行应用结构
- 是否有演示路径
- 是否有交付边界
- 是否可被 OpenYida 自动搭建

## 五、核心模块设计

### 5.1 AI 需求诊断器

输入：

- 客户名称
- 行业
- 客户规模
- 当前业务痛点
- 会议纪要或商机描述
- 已知钉钉/宜搭使用情况

输出：

- 需求摘要
- 业务场景分类
- 推荐方案 Top 3
- 推荐切入话术
- Demo 搭建建议
- 下一步客户问题清单

### 5.2 可执行方案包

每个方案不只是文章，而是一个可执行资产包。

| 资产 | 说明 |
| --- | --- |
| 方案概览 | 客户问题、价值主张、适用行业、客户规模 |
| 应用蓝图 | 表单、流程、报表、权限、连接器、数据模型 |
| 悟空 Prompt | 需求分析、行业改写、话术生成、异议处理、PRD 生成 |
| OpenYida 动作 | 创建应用、创建表单、创建流程、发布页面、生成报表 |
| 演示资产 | 演示路径、演示数据、客户版话术、FAQ |
| 交付资产 | 交付计划、验收清单、边界说明、风险提示 |

### 5.4 悟空 Prompt 资产库

Prompt 应和方案强绑定，不做孤立 Prompt 展示。

Prompt 分类：

- 客户需求分析 Prompt
- 行业方案生成 Prompt
- 场景改写 Prompt
- 演示话术 Prompt
- 客户异议处理 Prompt
- PRD 生成 Prompt
- 交付计划 Prompt
- 方案质检 Prompt

### 5.5 OpenYida 一键搭建

OpenYida 在中心里的角色：

**把 AI 生成的方案，变成宜搭里真实可运行的应用。**

第一阶段支持：

- 创建样板应用
- 创建核心表单
- 创建流程表单
- 创建自定义首页
- 创建经营看板
- 生成测试数据
- 输出应用地址和交付清单

第二阶段支持：

- 根据客户反馈增量改版
- 生成连接器模板
- 生成权限组
- 生成客户专属演示数据
- 回收 Demo 环境和版本归档

### 5.6 SA 客户拜访工作台

客户拜访模块服务单个 SA 的客户经营，不只记录拜访日程，还要把拜访前、中、后的 AI 动作串起来。

核心链路：

**拜访计划 → 客户画像 → 方案准备 → 现场记录 → 会后纪要 → 下一步推进 → 主管可见状态**

关键能力：

| 能力 | 说明 |
| --- | --- |
| 今日 / 本周拜访 | 展示待拜访客户、时间、阶段、目标和准备状态 |
| 拜访前准备 | AI 基于客户行业、商机阶段、历史记录推荐方案、Demo、话术 |
| 拜访纪要生成 | 从钉钉会议、闪记或手工记录生成客户需求摘要 |
| 客户状态更新 | 自动更新客户阶段、意向等级、风险、下一步动作 |
| 方案包绑定 | 每次拜访绑定推荐方案、客户材料、OpenYida Demo |
| 会后跟进 | 自动生成待办、客户问题清单、补充材料和下次拜访建议 |

拜访状态建议：

- 待准备
- 已准备
- 已拜访待纪要
- 待客户反馈
- 方案确认中
- Demo 已演示
- 转交付
- 暂缓 / 风险

拜访记录最小字段：

| 字段 | 说明 |
| --- | --- |
| 客户名称 | 客户主数据 |
| SA | 负责人 |
| 行业 | 行业作战地图入口 |
| 商机阶段 | 初访、需求确认、方案演示、商务推进、交付评估 |
| 拜访目标 | 本次拜访要推进什么 |
| 推荐方案 | 绑定方案包 |
| Demo 状态 | 未创建、已创建、已演示、需改版 |
| 下一步动作 | 会后必须有明确动作 |
| 风险标签 | 预算、决策链、竞品、交付边界、集成复杂度 |

### 5.7 SA 主管看板

主管看板服务 SA 老大，核心不是看静态报表，而是快速判断团队大盘、个人状态、客户风险和资源投入方向。

主管需要回答 5 个问题：

1. 现在大盘健康吗？
2. 哪些客户值得重点推进？
3. 哪些客户有风险？
4. 哪些 SA 需要支持？
5. 哪些方案和 Demo 正在真正带来复用？

建议看板结构：

| 模块 | 指标 |
| --- | --- |
| 大盘状态 | 活跃客户数、近 7 天拜访数、方案演示数、Demo 创建数、转阶段客户数 |
| 漏斗状态 | 初访、需求确认、方案演示、POC/Demo、商务推进、交付评估 |
| 个人状态 | 每个 SA 的客户数、拜访数、跟进逾期、Demo 数、方案演示数、风险客户数 |
| 客户风险 | 高价值但停滞、竞品介入、无下一步、Demo 未跟进、决策链不清 |
| 方案复用 | 方案被推荐次数、客户材料生成次数、Demo 创建次数、演示反馈 |
| 管理动作 | 需要主管协同拜访、需要行业专家支持、需要交付预评估 |

个人状态不建议只看拜访数量，应该看“推进质量”：

| 指标 | 解释 |
| --- | --- |
| 准备完成率 | 拜访前是否生成方案包、话术、客户材料 |
| 会后纪要率 | 拜访后是否沉淀需求和下一步动作 |
| 下一步明确率 | 客户是否有明确 owner、时间点和推进事项 |
| Demo 跟进率 | 创建 Demo 后是否完成演示、反馈和改版 |
| 风险处理率 | 风险客户是否被标记、升级和处理 |

主管看板可以给出 AI 管理建议：

- 本周建议主管陪访的 Top 5 客户
- 需要行业专家支持的客户
- 已停滞超过 14 天的高价值客户
- Demo 创建后 7 天未演示的客户
- 方案演示后无下一步的客户
- 每位 SA 的下周重点动作

## 七、方案成熟度评分

每个方案进入中心时自动生成评分。

| 维度 | 权重 | 说明 |
| --- | ---: | --- |
| 行业匹配度 | 20 | 是否有明确行业、客户类型、痛点 |
| 业务价值 | 20 | 是否能讲清 ROI、效率、风险、体验价值 |
| 可演示度 | 20 | 是否有 Demo、演示路径、演示数据 |
| 可交付度 | 20 | 是否有应用蓝图、交付清单、边界说明 |
| AI 化程度 | 10 | 是否有悟空 Prompt 和 AI 助理流程 |
| OpenYida 覆盖度 | 10 | 是否支持一键搭建或半自动搭建 |

评分结果：

- S 级：可直接客户演示和复制交付
- A 级：可演示，需要少量客户化
- B 级：可参考，需要补齐 Demo 或交付资产
- C 级：仅概念，需要行业专家完善

## 八、MVP 范围

第一版只做三个闭环。

### 7.1 闭环一：客户需求到推荐方案

SA 输入客户需求，AI 输出推荐方案 Top 3，并说明推荐理由、适用边界和下一步动作。

### 7.2 闭环二：方案到可演示 Demo

SA 选择方案，OpenYida 创建样板应用或生成搭建任务清单。

### 7.3 闭环三：方案到客户材料

SA 一键生成客户版方案摘要、演示话术、FAQ、交付清单。

### 7.4 闭环四：拜访到团队经营

SA 每次拜访都沉淀客户状态、方案包、Demo 状态和下一步动作，主管看板自动汇总团队大盘和个人状态。

## 十一、衡量指标

| 指标 | 目标 |
| --- | --- |
| SA 方案准备时间 | 从小时级降到 15 分钟内 |
| 可一键搭建方案数 | 第一阶段不少于 10 个 |
| 方案复用次数 | 每周持续增长 |
| Demo 创建成功率 | 大于 90% |
| 客户材料生成满意度 | 大于 4/5 |
| 共创方案审核通过率 | 持续提升 |
| 拜访前准备完成率 | 大于 85% |
| 会后纪要沉淀率 | 大于 90% |
| 下一步明确率 | 大于 80% |
| 主管风险客户识别率 | 大于 90% |

## 十二、风险与约束

| 风险 | 应对 |
| --- | --- |
| 方案质量不稳定 | 引入成熟度评分和专家审核 |
| AI 输出过泛 | 强制方案绑定行业、场景、Demo、交付资产 |
| OpenYida 自动搭建覆盖不足 | 先覆盖标准表单、流程、报表、自定义页 |
| SA 不愿维护资产 | 用复用次数、推荐位和优秀方案激励 |
| 客户材料泄露内部信息 | 区分内部版和客户版，一键脱敏 |
