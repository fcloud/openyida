# 钉钉流程管理技能 - 使用示例

## 场景一：查询待处理流程

### 用户提问
```
我的流程有哪些？
```

### 助手响应
```
师父，您目前有 5 个待处理流程：

【待办事项】
1. [周报提交] - 系统提醒 - 2026-03-23 08:00 (ID: TODO_001, 类型：待办事项)
2. [会议预定确认] - 马骁尧 - 2026-03-23 07:30 (ID: TODO_002, 类型：待办事项)

【待审批流程】
3. [请假申请 - 张三] - 张三 - 2026-03-22 18:00 (ID: APPROVAL_001, 类型：待审批)
4. [报销审批 - 王五] - 王五 - 2026-03-22 17:30 (ID: APPROVAL_002, 类型：待审批)
5. [采购申请 - 李四] - 李四 - 2026-03-22 16:00 (ID: APPROVAL_003, 类型：待审批)

您可以说"第 X 个，同意/拒绝"来快速处理。
```

## 场景二：简单审批指令

### 用户提问
```
第三个，同意
```

### 助手响应
```
好的师父，已同意第 3 个流程"[请假申请 - 张三]"。

✅ 审批成功
- 流程：请假申请
- 发起人：张三
- 您的操作：同意
- 时间：2026-03-23 09:26
```

## 场景四：多种指令格式

以下指令格式都被支持：

```
✓ "第二个，同意"
✓ "第一个 拒绝"
✓ "同意第三个"
✓ "拒绝 第二个"
✓ "第五个，通过"
✓ "批准第一个"
✓ "驳回第三个，信息不完整"
```

## 场景五：错误处理

### 用户提问（序号超出范围）
```
第十个，同意
```

### 助手响应
```
师父，当前只有 5 个待处理流程，请输入 1-5 之间的序号。

您可以重新说"第 X 个，同意/拒绝"，其中 X 为 1 到 5 之间的数字。
```

### 用户提问（操作不明确）
```
第三个，看一下
```

### 助手响应
```
师父，请明确您的操作：同意 或 拒绝。

例如：
- "第三个，同意"
- "第三个，拒绝"
- "同意第三个"
```

```
助手：✅ 已同意第 1 个流程

[显示剩余流程]
```

## 最佳实践建议

1. **统一编号**：待办和待审批合并显示，方便用户说"第几个"
2. **清晰反馈**：每次审批后显示明确的执行结果
3. **容错处理**：序号错误时提示有效范围
4. **灵活指令**：支持多种自然语言表达方式
5. **状态同步**：处理完后主动询问是否继续处理其他流程
