# 钉钉流程管理技能 - 演示脚本

## 🎯 演示目标

展示如何通过自然语言快速查询和处理钉钉待办、待审批流程。

---

## 📖 演示场景

### 场景一：初次使用 - 查询流程

**演示者操作**：
```
我的流程有哪些？
```

**预期响应**：
```
师父，您目前有 5 个待处理流程：

【待办事项】
1. [周报提交] - 系统提醒 - 2026-03-23 08:00
2. [会议预定确认] - 马骁尧 - 2026-03-23 07:30

【待审批流程】
3. [请假申请 - 张三] - 张三 - 2026-03-22 18:00
4. [报销审批 - 王五] - 王五 - 2026-03-22 17:30
5. [采购申请 - 李四] - 李四 - 2026-03-22 16:00

您可以说"第 X 个，同意/拒绝"来快速处理。
```

**演示要点**：
- ✅ 待办和待审批合并显示
- ✅ 统一编号，方便引用
- ✅ 显示关键信息（标题、发起人、时间）
- ✅ 提示用户如何操作

---

### 场景二：快速审批

**演示者操作**：
```
第三个，同意
```

**预期响应**：
```
好的师父，已同意第 3 个流程"[请假申请 - 张三]"。

✅ 审批成功
- 流程：请假申请
- 发起人：张三
- 您的操作：同意
- 时间：2026-03-23 09:26
```

**演示要点**：
- ✅ 自然语言指令识别准确
- ✅ 执行结果清晰反馈
- ✅ 包含完整审批信息

---

### 场景三：带备注的拒绝

**演示者操作**：
```
第四个，拒绝，需要补充发票明细
```

**预期响应**：
```
好的师父，已拒绝第 4 个流程"[报销审批 - 王五]"。

❌ 已驳回
- 流程：报销审批
- 发起人：王五
- 您的操作：拒绝
- 备注：需要补充发票明细
- 时间：2026-03-23 09:26
```

**演示要点**：
- ✅ 支持复杂指令（带备注）
- ✅ 备注信息正确提取
- ✅ 拒绝操作明确标识

---

### 场景四：多种指令格式

**演示者依次输入**：
```
第一个 同意
同意第二个
批准第五个
驳回第三个，信息不完整
```

**预期响应**：
每次都能正确识别并执行对应操作。

**演示要点**：
- ✅ 支持多种表达方式
- ✅ 同义词识别（同意/通过/批准）
- ✅ 灵活的自然语言理解

---

### 场景五：错误处理

**演示者操作**：
```
第十个，同意
```

**预期响应**：
```
师父，当前只有 5 个待处理流程，请输入 1-5 之间的数字。

您可以重新说"第 X 个，同意/拒绝"，其中 X 为 1 到 5 之间的数字。
```

**演示要点**：
- ✅ 序号范围验证
- ✅ 友好的错误提示
- ✅ 引导用户正确操作

---

### 场景六：批量处理工作流

**完整对话演示**：

```
用户：我的流程有哪些？
助手：[显示 5 个流程]

用户：第一个，同意
助手：✅ 已同意

用户：第二个，同意
助手：✅ 已同意

用户：第三个，拒绝，需要补充材料
助手：❌ 已拒绝

用户：还剩几个？
助手：师父，您还有 2 个待处理流程：
      [显示剩余流程]
```

**演示要点**：
- ✅ 连续操作流畅
- ✅ 状态实时更新
- ✅ 支持追问

## 💡 演示技巧

1. **对比法**：先展示传统方式的繁琐，再展示新技能的便捷
2. **场景化**：用真实的业务场景（请假、报销、采购）引起共鸣
3. **互动性**：邀请观众提出指令，现场测试
4. **数据化**：强调效率提升（从 3 分钟到 3 秒）
5. **可视化**：投屏展示完整的对话过程

---

## 🎯 核心价值主张

演示结束时强调这三点：

1. **效率提升**：从"点击进入→查找→审批"简化为"一句话"
2. **体验优化**：自然语言交互，零学习成本
3. **智能集成**：无缝对接钉钉现有流程，无需改变习惯
