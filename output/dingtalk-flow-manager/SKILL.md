# 钉钉流程管理技能

## 核心功能

本技能用于帮助用户快速查询和处理钉钉上的待办事项与待审批流程。

### 触发场景

- 用户询问"我的流程有哪些"、"有什么待办"、"待审批的事项"等
- 用户需要查看钉钉待办事项列表
- 用户需要查看钉钉待审批流程列表
- 用户通过自然语言指令执行审批操作（如"第二个，同意"、"第一个，拒绝并备注需要补充材料"）

### 工作流程

#### 1. 查询流程列表

当用户询问流程时：

1. 调用 `dws` 技能的待办查询能力，获取用户的待办事项列表
2. 调用 `dws` 技能的审批查询能力，获取用户的待审批流程列表
3. **合并两个列表**，按时间顺序或优先级排序，统一编号显示
4. 向用户展示合并后的列表，格式如下：

```
师父，您目前有 X 个待处理流程：

【待办事项】
1. [标题] - [发起人] - [时间]
2. [标题] - [发起人] - [时间]

【待审批流程】
3. [标题] - [发起人] - [时间]
4. [标题] - [发起人] - [时间]

您可以说"第 X 个，同意/拒绝"来快速处理。
```

#### 2. 执行审批操作

当用户发出指令如"第二个，同意"或"第三个，拒绝"时：

1. 解析用户指令，提取：
   - 序号（第几个）
   - 操作类型（同意/拒绝）
   - 可选的备注信息
2. 根据序号定位到对应的流程项
3. 调用 `dws` 技能的审批执行能力，传入：
   - 流程 ID
   - 操作类型（同意/拒绝）
   - 备注信息（如有）
4. 向用户确认操作结果

### 自然语言指令格式

支持以下指令格式：

- `{序号}，{操作}` - 例如："第二个，同意"
- `{序号} {操作}` - 例如："第一个 拒绝"
- `{操作} 第{序号}个` - 例如："同意第三个"
- `{操作} {序号}` - 例如："拒绝 第二个"

支持的操作用词：
- 同意类：同意、通过、批准、OK、好的、可以
- 拒绝类：拒绝、驳回、不通过、否决

### 调用 dws 技能

本技能依赖 `dws` (钉钉工作台) 技能来实际执行钉钉 API 调用。

**查询待办：**
```
使用 dws 技能查询当前用户的待办事项列表
```

**查询待审批：**
```
使用 dws 技能查询当前用户的待审批流程列表
```

**执行审批：**
```
使用 dws 技能执行审批操作，传入流程 ID、操作类型和备注
```

### 错误处理

- 如果序号超出范围，提示用户有效的序号范围
- 如果钉钉 API 调用失败，告知用户具体错误信息
- 如果流程已被他人处理，提示用户流程状态已变更

### 示例对话

**示例 1：查询流程**
```
用户：我的流程有哪些？
助手：师父，您目前有 3 个待处理流程：

【待办事项】
1. 请假申请 - 张三 - 2026-03-23 09:00
2. 会议预定 - 李四 - 2026-03-23 08:30

【待审批流程】
3. 报销审批 - 王五 - 2026-03-22 18:00

您可以说"第 X 个，同意/拒绝"来快速处理。
```

**示例 2：执行审批**
```
用户：第三个，同意
助手：好的师父，已同意第 3 个流程"报销审批 - 王五"。
```

**示例 3：带备注的拒绝**
```
用户：第一个，拒绝，需要补充证明材料
助手：好的师父，已拒绝第 1 个流程"请假申请 - 张三"，备注：需要补充证明材料。
```
