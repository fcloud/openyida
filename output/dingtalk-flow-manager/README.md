# 钉钉流程管理技能 (dingtalk-flow-manager)

## 技能说明

这是一个用于管理钉钉待办事项和待审批流程的技能。当用户询问"我的流程有哪些"时，会自动列出所有待处理流程；当用户说"第二个，同意"时，会自动执行审批操作。

## 目录结构

```
dingtalk-flow-manager/
├── SKILL.md                    # 技能主文件（包含触发条件和工作流程）
├── README.md                   # 本说明文件
├── scripts/
│   └── query_flows.py         # 流程查询和处理脚本
├── references/
│   ├── api_reference.md       # 钉钉 API 参考文档
│   └── usage_examples.md      # 使用示例
└── assets/                     # 资源目录（预留）
```

## 核心功能

1. **合并显示**：将待办事项和待审批流程合并成一个统一列表，按时间排序
2. **自然语言指令**：支持多种表达方式（如"第二个，同意"、"同意第三个"等）
3. **自动执行**：解析用户指令后自动调用钉钉 API 执行审批
4. **错误处理**：序号验证、权限检查、状态同步

## 使用方法

### 在悟空平台中使用

1. **上传技能包**：
   - 将整个 `dingtalk-flow-manager` 文件夹打包成 ZIP
   - 在悟空平台的技能管理中上传

2. **启用技能**：
   ```bash
   real_cli skills enable dingtalk-flow-manager
   ```

3. **开始使用**：
   - 问："我的流程有哪些？"
   - 说："第二个，同意"

### 依赖技能

- **dws (钉钉工作台)**：实际执行钉钉 API 调用

## 支持的指令格式

```
✓ "第 X 个，同意/拒绝"
✓ "第 X 个 同意/拒绝"
✓ "同意/拒绝 第 X 个"
✓ "同意/拒绝 X"
```

示例：
- "第二个，同意"
- "第一个 拒绝"
- "同意第三个"
- "拒绝 第二个，需要补充材料"

## 开发说明

### 脚本功能

`scripts/query_flows.py` 提供以下功能：

- `merge_and_sort_flows()`: 合并待办和待审批流程
- `format_flow_item()`: 格式化流程项显示
- `parse_user_command()`: 解析自然语言指令

### 扩展建议

如需增强此技能，可以考虑：

1. **添加筛选功能**：按类型、发起人、时间范围筛选
2. **批量操作**：支持"全部同意"、"同意前三个"等指令
3. **提醒功能**：定时提醒未处理的紧急流程
4. **统计分析**：每周/月审批统计报告

## 注意事项

1. 首次使用需要授权访问钉钉待办和审批数据
2. 确保 `dws` 技能已启用
3. 流程状态实时同步，但可能有 1-2 秒延迟
4. 已被他人处理的流程会显示状态变更提示

## 测试建议

在正式使用前，建议测试以下场景：

- [ ] 查询空列表（没有待处理流程）
- [ ] 查询单个流程
- [ ] 查询多个流程（混合待办和待审批）
- [ ] 同意操作
- [ ] 拒绝操作（带备注）
- [ ] 序号超出范围
- [ ] 指令格式识别（多种表达）
- [ ] 并发处理（多人同时审批同一流程）
