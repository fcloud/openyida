# yida-flash-note-to-prd — 闪记转高质量 Prompt 技能

将钉钉闪记（会议纪要、AI 听记）自动转化为结构化 prompt，写入 `prd/<项目名>.md`，可直接驱动后续宜搭应用开发流程。

> **核心理念**：`prd/<项目名>.md` 不是传统产品文档，而是一段**精心结构化的 prompt**——包含功能模块、字段设计、流程设计、报表需求等业务语义，使后续技能可直接读取执行。

## 核心规则

**致命（FATAL）**：
1. **无闪记内容不得生成 PRD**：必须等待用户提供闪记文本、图片或链接
2. **输出路径固定**：必须写入 `prd/<项目名>.md`，不得写入项目根目录
3. **输出必须是结构化 prompt**：不得生成传统产品文档格式

**重要（IMPORTANT）**：
1. **自动识别输入类型**：文本 / 图片 / 闪记链接，分别处理
2. **A1 摘要最高优先级**：不得忽略闪记中的结构化摘要（📋 会议摘要、✅ 待办事项、📌 决策结论）
3. **提取会议元信息**：标题、参会人、时间、时长等，写入 prompt 上下文
4. **PRD 必须完整**：功能模块、字段设计表、流程设计、报表需求、待确认事项缺一不可
5. **生成后必须确认**：向用户展示摘要，确认后再进入开发流程
6. **字段类型规范**：必须使用宜搭标准字段类型，详见 [yida-field-types.md](references/yida-field-types.md)

## 适用场景 / 触发条件

**正向触发**（自动识别，无需用户明确说"转 PRD"）：
- 用户发送闪记文本 / 图片（钉钉闪记截图）
- 用户提到"闪记"、"会议纪要"、"会议记录"、"闪记转需求"
- 用户发送钉钉闪记链接（`shanji.dingtalk.com` 或 `alidocs.dingtalk.com`）
- 用户口述会议要点，要求整理成需求

**不适用场景（不要触发）**：
- 直接创建宜搭应用 → PRD 生成后还需 `yida-app` 流程
- 处理非需求类文档（技术文档、代码文件）
- 用户已有完整 PRD → 直接使用 `yida-app`

## 输入方式

### 文本输入
用户直接粘贴闪记文本，进入预处理和 prompt 生成流程。

### 图片输入
1. 使用视觉能力**完整识别**图片中所有文字（标题、正文、表格、列表）
2. 保留结构化布局的逻辑关系（分栏、卡片、流程图）
3. 识别标签/标注（Emoji、高亮文字等重点信息）
4. 识别结果作为闪记文本继续处理

### 闪记链接
URL 模式：`https://shanji.dingtalk.com/app/transcribes/...` 或 `https://alidocs.dingtalk.com/...`
闪记页面为 SPA 应用需登录态，处理方式：提示用户在钉钉中打开链接，复制全文后粘贴（或导出 .txt 文件）。

### 会议识别能力

预处理完成后自动执行以下识别：

**会议元信息**：从闪记头部提取标题、时间、时长、参会人、主持人、地点（匹配关键词如"会议主题"、"参会人员"等）。

**钉钉 A1 摘要识别**：自动识别 AI 生成的结构化摘要段落：

| 段落类型 | 标识（Emoji / 纯文本） |
|---------|---------------------|
| 会议摘要 | 📋 📝 / 【会议摘要】 |
| 待办事项 | ✅ ☑️ / 【待办事项】 |
| 决策结论 | 📌 🔑 💡 / 【决策结论】 |
| 讨论要点 | 📊 📈 / 【讨论要点】 |
| 后续跟进 | ⏭️ 👉 / 【后续跟进】 |

> A1 摘要段落为高优先级信息，在 Prompt 中优先注入。

**发言人角色识别**：支持 `张三(产品经理)：`、`李四（技术负责人）：`、`王五[设计师]：` 等格式，角色信息注入 Prompt 帮助区分不同视角。

## AI 智能体自动执行流程

> **重要**：主要执行方式是 AI 智能体直接处理（自身即大模型），而非调用 CLI 命令。

AI 执行时自动按三层 Prompt 架构处理：**识别输入 → 预处理 → 生成结构化 prompt → 写入 `prd/<项目名>.md` → 输出摘要并确认**。
每一步的核心提取规则、质量标准和分段策略详见 [Prompt 架构](references/flash-note-prompt.md)。

### 闪记预处理规则（在 Step 1 之前执行）

| 处理步骤 | 说明 | 正则/规则 |
|---------|------|----------|
| 去除时间戳 | 移除 `[00:01:23]` 等时间标记 | `/\[?\d{1,2}:\d{2}(?::\d{2})?\]?\s*/g` |
| 去除口语填充词 | 移除"嗯"、"那个"、"就是说"等 | 预定义填充词列表匹配 |
| 合并连续空行 | 多个空行合并为单个换行 | `/\n{3,}/g` → `\n\n` |
| 去除重复标点 | 连续重复标点合并 | `/([，。！？,\.!?])\1+/g` |
| 合并连续发言 | 同一人连续多段合并（逗号连接） | 按发言人前缀匹配 |
| 保留发言人标识 | 保留"张三："等前缀 | 支持中英文冒号格式 |

### 长文本分段处理

- ≤ 6000 字 → 单段模式：三层 Prompt 架构，直接输出 Markdown PRD
- \> 6000 字 → 按段落边界智能分段，每段输出 JSON 结构化数据，最终合并去重生成完整 PRD

## 命令 & 参数（可选的自动化工具）

> CLI 命令适用于批量处理或脚本化场景，日常使用直接将闪记发给 AI 智能体即可。

```bash
openyida flash-to-prd --file <闪记文件路径> [--name <项目名>] [--max-tokens <8000>]
cat meeting.txt | openyida flash-to-prd --name <项目名>
```

| 参数 | 必填 | 说明 |
|------|------|------|
| `--file` | 否 | 闪记文本文件路径（.txt / .md） |
| `--name` | 否 | 项目名称（默认从闪记内容提取） |
| `--max-tokens` | 否 | AI 最大输出 token 数（默认 8000） |

**前置依赖**：Node.js ≥ 16，已登录宜搭（`.cache/cookies.json` 有效）。

## 异常处理 / Agent 错误处理策略

| 错误类型 | 处理策略 |
|---------|----------|
| 输入内容为空 | 停止执行，主动询问用户提供闪记内容（文本/图片/链接） |
| 输入过短（<100 字） | 提示内容可能不足，询问是否继续或补充 |
| 图片识别失败 | 提示重新上传清晰图片，或改用文本粘贴 |
| 闪记链接无法访问 | 引导用户在钉钉中打开链接，复制全文后粘贴 |
| 文件写入失败 | 检查目录权限，尝试创建 `prd/` 目录后重试 |
| 生成结果不完整/质量不佳 | 展示已生成内容和待确认事项，引导用户补充关键信息后重新生成 |
| AI 接口调用失败 | 确认登录态有效（`.cache/cookies.json`），重新执行 |
| 未知错误 | 停止执行，完整展示错误信息，建议用户反馈问题 |

## 跨技能协作

PRD 生成后可无缝衔接宜搭开发流程：

| 后续步骤 | 对应技能 | 从 PRD 获取 |
|---------|---------|------------|
| 创建应用 | `yida-create-app` | 项目名称 |
| 创建表单 | `yida-create-form-page` | 字段设计（名称、类型、必填、选项） |
| 配置流程 | `yida-create-process` | 流程设计（节点、审批人、条件） |
| 自定义页面 | `yida-custom-page` | 功能模块、页面布局需求 |
| 创建报表 | `yida-report` | 数据报表需求 |

## 注意事项

1. **闪记质量影响 PRD 质量**：碎片化或口语化严重的闪记，PRD 可能需用户补充完善
2. **待确认事项必须处理**：PRD 中的"待确认事项"需用户逐一确认后才能进入开发
3. **PRD 不记录技术 ID**：只记录业务语义信息，`formUuid`、`fieldId` 等写入 `.cache/`
4. **支持迭代优化**：用户可对 PRD 提出修改意见，AI 基于反馈重新调整
5. **本技能不读写 memory**：输出到 `prd/<项目名>.md` 文件，不依赖跨会话状态
