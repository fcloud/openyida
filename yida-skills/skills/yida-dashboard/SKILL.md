# 宜搭 Dashboard 看板技能

## 定位

本技能负责完整看板场景的产品化交付，不只是画图。

`yida-dashboard` 负责：页面结构、数据接入、筛选、响应式、截图、短链、导航隐藏、待办闭环、发布验证。

`yida-chart` 负责：具体 ECharts 图表方案、图表配置、图表渲染细节。

`yida-report` 负责：宜搭原生报表和服务端聚合数据源。

## 严格要求

- 默认使用组织内短链接，不默认公开访问。
- 默认隐藏宜搭左侧导航和顶部突兀搜索入口，除非用户明确要求保留。
- 默认多端友好，PC、窄屏、移动端都要可读可操作。
- 默认接真实表单、流程或报表数据；没有数据时先说明并用明确的演示数据兜底，不伪装成真实数据。
- 默认提供日期、财年和业务维度筛选；筛选项必须从数据或字段枚举中推导。
- 默认支持卡片级截图，截图当前卡片区域，不截整页或错误区域。
- 涉及客户、人名、项目名等隐私信息时必须脱敏，不写入类似真实客户代号的硬编码样例。
- 发布前必须运行 `openyida check-page` 和 `openyida compile`。

## 禁止事项

- 不要把看板做成营销落地页或说明页，第一屏必须是可用看板。
- 不要只生成静态漂亮页面后宣称已接数据。
- 不要在前端硬编码钉钉 token、AppSecret 或个人凭证。
- 不要从 `example.test`、`localhost` 等占位地址取业务数据。
- 不要把负责人输入框做成假搜索；必须能搜索并选择真实组织成员，或明确说明未接通。
- 不要让“创建钉钉待办”只停留在前端 toast；必须写入触发表单并配置自动化，或说明还缺连接器/凭证。

## 标准工作流

1. 确认对象
   - 已有应用：确认 `appType`、看板页面 `formUuid`、相关数据表单/流程/报表。
   - 新建应用：先走 `yida-app` / `yida-create-app` / `yida-create-page`。
   - 先执行 `openyida env --json` 和 `openyida login --check-only --json`，确认登录态和组织。

2. 盘数据
   - 用 `yida-get-schema` 获取字段 ID、字段类型和枚举。
   - 用 `yida-data-management` 抽样查询数据，判断可用字段、日期字段、财年口径、维度字段和数据量。
   - 聚合统计优先用 `yida-report` 或报表数据接口；小数据量明细可直接查询表单。

3. 设计默认筛选
   - 必备：日期范围或财年。
   - 常见维度：区域、品类、客户类型、状态、负责人、贸易方向、流程状态。
   - 默认值从数据推导，例如最新财年、当前月、全部维度。

4. 搭页面
   - 用 `yida-custom-page` 编写 React 16 类组件模式页面。
   - 顶部只保留业务标题、关键状态、时间或组织内标识，不放突兀全局搜索。
   - 第一屏包含核心指标、经营动作、风险水位、关键图表或列表。
   - 图表需要 ECharts 时调用 `yida-chart`，通过 `this.utils.loadScript` 从 `g.alicdn.com` 加载。

5. 截图能力
   - 默认使用 `html2canvas`：
     `https://g.alicdn.com/code/lib/html2canvas/1.4.1/html2canvas.min.js`
   - 截图按钮只放在具体卡片或图表卡右上角。
   - 截图目标必须是卡片 DOM 节点本身，设置 `width/height/scrollX/scrollY`，忽略按钮本身。
   - 截图失败要 toast 提示，不静默失败。

6. 钉钉待办/通知闭环
   - 前端先写入“待办触发表单”，字段至少包含标题、负责人 EmployeeField、优先级、截止时间、说明、状态。
   - 负责人搜索使用同源接口：
     `/query/userservice/searchUserList.json?appType=<appType>&key=<keyword>&orderNum=`
   - 解析真实返回结构时优先读取 `content.userList`，并兼容旧结构。
   - 输入负责人后必须从搜索结果中选择，不能把手输姓名当成 userId。
   - 用 `yida-integration` 创建自动化，把触发表单新增事件通知给负责人字段。
   - 如果用户要求真正的钉钉 Todo 2.0 任务本体，使用 `yida-connector` 或服务端连接器接钉钉开放接口，不在自定义页面前端保存密钥。

7. 发布配置
   - `openyida check-page <source>`
   - `openyida compile <source>`
   - `openyida publish <source> <appType> <formUuid>`
   - 用 `yida-page-config` 或发布结果设置组织内短链接。
   - 用 `update-form-config` 或页面配置能力隐藏导航。

8. 验证
   - 短链能打开，导航隐藏。
   - 数据来自真实表单/报表，筛选生效。
   - 移动端和 PC 无明显溢出、重叠、字号失控。
   - 截图截到当前卡片区域。
   - 负责人搜索返回候选并能选择。
   - 创建待办后，触发表单新增记录；自动化流启用并绑定负责人字段。

## 默认交付说明

完成后向用户说明：

- 看板短链。
- 数据源来自哪些表单/报表。
- 已启用的筛选维度。
- 截图库和 CDN。
- 钉钉待办/通知闭环状态。
- 已运行的校验命令。
